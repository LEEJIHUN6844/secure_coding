# 보고서 개요 (PDF 작성용 골격)

제출 파일명: `[WHS][secure-coding][XX반]이름(전화번호뒷자리).pdf` (공백 없음)

## 1. 개요 및 목표
- 중고거래가 가능한 플랫폼을 SDLC 전 과정에 따라 개발
- 전 과정에서 보안 약점을 선제적으로 제거하는 것이 목표

## 2. 요구사항 분석
- 기능 요구사항 7가지 (슬라이드 기준) 표로 정리
- 직접 분석한 3가지(송금/검색/관리자):
  - 송금: 실결제 대신 가상 포인트 방식 채택 이유, 잔액 검증, 원자성
  - 검색: 상품명 키워드 검색, 입력 길이 제한
  - 관리자: role 기반 권한, 유저 휴면/상품 차단/신고 열람
- 비기능 요구사항: 보안(핵심), 사용성, 성능

## 3. 시스템 설계
- 아키텍처: Flask(백엔드) + Jinja2(템플릿) + Socket.IO(실시간) + SQLite
- DB 설계 (ERD): user / product / report / dm_thread / message / transfer
- 페이지 설계: 목록+전체채팅, 회원가입, 로그인, 마이페이지, 상품 등록/상세/수정,
  1:1 채팅, 신고, 지갑, 관리자 대시보드
- 위협 모델링: 기능별 예상 공격 표 (SQLi, XSS, CSRF, IDOR, 무차별 대입, 파일 업로드, 경쟁 조건)

각 항목을 "공격 시나리오 → 취약한 코드(Before) → 조치한 코드(After)" 형식으로 작성.
docs/security-checklist.md 의 표를 기반으로 작성.

## 6. 체크리스트 및 테스팅
- 기능 체크리스트 + 보안 체크리스트 (통과 여부 표)
- 실제 공격 테스트 결과 (스크린샷 포함 권장):
  - `' OR 1=1--` 로그인 시도 → 실패 확인
  - `<script>alert(1)</script>` 상품명/채팅 → 이스케이프 확인
  - 타인 상품 수정 URL 직접 접근 → 403 확인
  - CSRF 토큰 제거 후 POST → 400 확인
  - 음수/초과 금액 송금 → 거부 확인
  - 로그인 5회 실패 → 잠금 확인

## 7. 유지보수

  (레이트리미터·소켓 세션 공유), 감사 로그, 비밀번호 재설정 등
